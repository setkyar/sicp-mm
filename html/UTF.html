Unofficial Texinfo Format

ဤသည်မှာ Unofficial Texinfo Format မှ SICP ဒုတိယအကြိမ်ထုတ် စာအုပ်ဖြစ်သည်။

သင်သည် ဤစာအုပ်ကို Emacs ၏ Info mode ကဲ့သို့သော Info hypertext browser တစ်ခုတွင် ဖတ်ရှုနေခြင်း ဖြစ်နိုင်ပါသည်။ သို့မဟုတ်ပါက သင်သည် သင်၏ screen သို့မဟုတ် printer ပေါ်တွင် TeX-formatted ဖြင့် ဖတ်ရှုနေခြင်းလည်း ဖြစ်နိုင်သော်လည်း ၎င်းမှာ မလိုအပ်သော အရာတစ်ခု ဖြစ်ပါလိမ့်မည်။ ထို့အပြင် print ထုတ်ပါကလည်း စရိတ်စက များပြားပါလိမ့်မည်။

အခမဲ့ ဖြန့်ဝေထားသော တရားဝင် HTML-and-GIF format ကို ၂၀၀၁ ခုနှစ်၊ ဧပြီလ၏ ရှည်လျားသော Emacs lovefest ပိတ်ရက်တစ်ခုအတွင်း Lytha Ayth က Unofficial Texinfo Format (UTF) version 1 အဖြစ်သို့ ကိုယ်တိုင် စတင် ပြောင်းလဲခဲ့ခြင်း ဖြစ်သည်။

UTF သည် HTML format ထက် ရှာဖွေရ ပိုမို လွယ်ကူပါသည်။ ၎င်းသည် လှူဒါန်းထားသော 386 အခြေခံ PC များကဲ့သို့သော အခြေခံ ကွန်ပျူတာများ အသုံးပြုသူများအတွက်လည်း ပိုမို လက်လှမ်းမီစေပါသည်။ 386 တစ်လုံးသည် သီအိုရီအရ Linux၊ Emacs နှင့် Scheme interpreter တို့ကို တစ်ပြိုင်နက်တည်း run နိုင်သော်လည်း 386 အများစုသည် Netscape နှင့် လိုအပ်သော X Window System နှစ်ခုလုံးကို run ရာတွင် ငွေကြေး မပြည့်စုံသော hacker လောင်းလျာများအား thrashing ဆိုသည့် သဘောတရားနှင့် မိတ်မဆက်ပေးမီမှာပင် အခက်တွေ့နိုင်ဖွယ် ရှိသည်။ UTF ကို compress မလုပ်ဘဲ 1.44MB floppy diskette တစ်ခုအတွင်း ထည့်သွင်းနိုင်ပြီး ၎င်းမှာ Internet သို့မဟုတ် LAN မရှိသော PC များတွင် UTF install လုပ်ရန်အတွက် အသုံးဝင်စေပါသည်။

Texinfo ပြောင်းလဲခြင်းသည် ဖြစ်နိုင်သမျှ အတိုင်းအတာအထိ တိုက်ရိုက် ပြောင်းလဲခြင်း (transliteration) သာ ဖြစ်သည်။ TeX-to-HTML ပြောင်းလဲခြင်း ကဲ့သို့ပင်၊ ဤပြောင်းလဲမှုတွင်လည်း ချို့ယွင်းချက်အချို့ ပါဝင်လာနိုင်ပါသည်။ Unofficial Texinfo Format တွင်မူ ရုပ်ပုံများသည် ပျောက်ကွယ်လုနီးပါး ဖြစ်နေသော ASCII အနုပညာ၏ အပျော်တမ်း ပြန်လည် ဆန်းသစ်မှုဒဏ်ကို ခံစားခဲ့ရသည်။ ထို့အပြင် မြောက်မြားစွာသော superscript (‘^’) နှင့် subscript (‘_’) များကို ပြောင်းလဲစဉ်အတွင်း ဝေဝါးသော အမှားအယွင်းအချို့ ပါဝင်သွားခြင်းလည်း ဖြစ်နိုင်ပါသည်။ မည်သည့်အရာက မှန်ကန်ကြောင်း ခန့်မှန်းရန် စာဖတ်သူအတွက် လေ့ကျင့်ခန်းအဖြစ် ထားရှိခဲ့ပါသည်။ သို့သော် အနည်းဆုံးတော့ ကျွန်ုပ်တို့သည် greater-than-or-equal သင်္ကေတကို <u>&gt;</u> အဖြစ် encode လုပ်ခြင်းဖြင့် ကျွန်ုပ်တို့၏ ရဲရင့်သော အာကာသယာဉ်မှူးများကို အန္တရာယ် မဖြစ်စေခဲ့ပါ။

အကယ်၍ သင်သည် အမှားများကို ပြင်ဆင်ရန် သို့မဟုတ် ASCII art ကို မြှင့်တင်ရန် sicp.texi ကို ပြုပြင်မွမ်းမံပါက သင် ပြုပြင်ခဲ့သည်များကို ဖော်ပြနိုင်ရန် @set utfversion 2.andresraba6.6 စာကြောင်းကို update လုပ်ပါ။ ဥပမာအားဖြင့် သင်သည် Lytha ၏ version 1 ဖြင့် စတင်ခဲ့ပြီး သင့်အမည်မှာ Bob ဖြစ်ပါက သင်၏ နောက်ဆက်တွဲ version များကို 1.bob1, 1.bob2, … 1.bobn ဟု အမည်ပေးနိုင်ပါသည်။ utfversiondate ကိုလည်း update လုပ်ပါ။ အကယ်၍ သင်သည် သင်၏ version ကို Web ပေါ်တွင် ဖြန့်ဝေလိုပါက ဖိုင် သို့မဟုတ် Web စာမျက်နှာ၏ တစ်နေရာရာတွင် “sicp.texi” ဆိုသည့် စာသားကို ထည့်သွင်းထားခြင်းဖြင့် လူများက Web search engine များဖြင့် ရှာဖွေရာတွင် ပိုမို လွယ်ကူစေပါလိမ့်မည်။

Unofficial Texinfo Format သည် စေတနာဖြင့် အခမဲ့ ဖြန့်ဝေထားသော HTML version ၏ ရည်ရွယ်ချက်နှင့် ကိုက်ညီသည်ဟု ယုံကြည်ရပါသည်။ သို့သော် ရှေ့နေ အုပ်စုကြီးတစ်ခုသည် မည်သည့်အချိန်တွင် တစ်စုံတစ်ခု လုပ်ဆောင်ရန် လိုအပ်လာပြီး အန္တရာယ်မရှိသော အသေးအမွှား ကိစ္စလေးတစ်ခုအပေါ်တွင် စိတ်ရှုပ်ထွေးနေမည်ကို သင် မသိနိုင်သောကြောင့် သင်၏ အမည်အပြည့်အစုံကို အသုံးမပြုမီ သို့မဟုတ် သင်၏ account သို့မဟုတ် စက်အမည် ပါဝင်နေနိုင်သော Info, DVI, PostScript သို့မဟုတ် PDF format များကို မဖြန့်ဝေမီ သေချာစွာ စဉ်းစားပါ။ Peath, Lytha Ayth

Addendum: Abelson နှင့် Sussman တို့၏ SICP ဗီဒီယို သင်ခန်းစာများကို MIT CSAIL သို့မဟုတ် MIT OCW တွင်လည်း ကြည့်ရှုနိုင်ပါသည်။

Second Addendum: အထက်ဖော်ပြပါအချက်မှာ ၂၀၀၁ ခုနှစ်မှ UTF ၏ မူလ နိဒါန်း ဖြစ်ပါသည်။ ဆယ်နှစ်အကြာတွင် UTF သည် ပြောင်းလဲတိုးတက်လာခဲ့သည်- သင်္ချာဆိုင်ရာ သင်္ကေတများနှင့် ပုံသေနည်းများကို စနစ်တကျ ပုံနှိပ်ထုတ်ဝေနိုင်ခဲ့ပြီး ရုပ်ပုံများကိုလည်း vector graphics ဖြင့် ရေးဆွဲခဲ့သည်။ မူလ စာသား ပုံသေနည်းများနှင့် ASCII art ရုပ်ပုံများသည် Texinfo source တွင် ရှိနေဆဲဖြစ်သော်လည်း Info output အဖြစ် compile လုပ်သောအခါမှသာ ပြသမည်ဖြစ်သည်။ e-book reader များနှင့် tablet များ ပေါ်ထွန်းလာချိန်တွင် screen ပေါ်၌ PDF ဖတ်ခြင်းသည် တရားဝင်အားဖြင့် မလိုအပ်သော အရာ မဟုတ်တော့ပါ။ ပျော်ရွှင်စွာ ဖတ်ရှုနိုင်ပါစေ! A.R, မေလ၊ ၂၀၁၁

Third Addendum: အရာရာသည် လည်ပတ်နေသည်။ ကျွန်ုပ်တို့သည် PDF စာအုပ်ကို ပုံနှိပ်ထုတ်ဝေရာတွင် Texinfo နှင့် LaTeX တို့၏ စွမ်းဆောင်ရည်များကို စူးစမ်းလေ့လာခဲ့ကြသည်။ ယခုအခါ HTML သို့ ပြန်လည် လာရမည့် အချိန်ဖြစ်သည်။ ယခုတစ်ကြိမ်တွင်မူ HTML5 ဖြစ်လာမည်ဖြစ်သည်။ “၎င်းသည် အန္တရာယ်ရှိသော နေရာတစ်ခု ဖြစ်နိုင်သော်လည်း ငြိမ်းချမ်းရေးအတွက် ကျွန်ုပ်တို့၏ နောက်ဆုံးနှင့် အကောင်းဆုံး မျှော်လင့်ချက် ဖြစ်သည်။” (Sinclair, Babylon 5။) browser စစ်ပွဲများအပြီးတွင် ငြိမ်းချမ်းရေးသည် တည်တံ့ပါမည်လား? မည်သို့ပင်ဆိုစေ၊ ဤပရောဂျက်၏ ရည်ရွယ်ချက်မှာ HTML5 toolbox ကို အသုံးပြု၍ LaTeX output ၏ အရည်အသွေးကို နီးစပ်စေရန်ဖြစ်သည်။ JavaScript, CSS3, SVG, MathJax, MathML နှင့် web font များကို ဤနေရာတွင် အသုံးပြုထားသည်။ ဤလုပ်ဆောင်ချက်၏ စမ်းသပ်မှု သဘောသဘာဝကြောင့် browser အချို့တွင် အမှားအယွင်းများ ရှိနိုင်ပါသည်။ နည်းပညာပိုင်းနှင့် အနုပညာပိုင်းဆိုင်ရာ အကူအညီများကို ကျေးဇူးတင်စွာ လက်ခံပါသည်။ ထိုအတောအတွင်း စာအုပ်ကို ပျော်ရွှင်စွာ ဖတ်ရှုပြီး REPL တစ်ခုကို စတင်လိုက်ပါ။ A.R, ဇန်နဝါရီလ၊ ၂၀၁၄
