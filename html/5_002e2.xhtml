5.2Register-Machine Simulator တစ်ခု

Register machine များ၏ ဒီဇိုင်းကို ကောင်းစွာနားလည်သဘောပေါက်ရန်အတွက် ကျွန်ုပ်တို့ဒီဇိုင်းထုတ်ထားသော machine များသည် မျှော်မှန်းထားသည့်အတိုင်း လုပ်ဆောင်ခြင်း ရှိ၊ မရှိ စမ်းသပ်ကြည့်ရပါမည်။ ဒီဇိုင်းတစ်ခုကို စမ်းသပ်ရန် နည်းလမ်းတစ်ခုမှာ Exercise 5.5 တွင်ပါရှိသည့်အတိုင်း controller ၏ လုပ်ဆောင်ချက်ကို လက်ဖြင့် လိုက်လံတွက်ချက်ခြင်း (hand-simulate) ဖြစ်သည်။ သို့သော် အလွန်ရိုးရှင်းသော machine များမှလွဲ၍ ကျန် machine များအတွက် ဤသို့လုပ်ဆောင်ရန်မှာ အလွန်ငြီးငွေ့ဖွယ်ကောင်းပါသည်။ ဤကဏ္ဍတွင် ကျွန်ုပ်တို့သည် register-machine language ဖြင့် ဖော်ပြထားသော machine များအတွက် simulator တစ်ခုကို တည်ဆောက်ပါမည်။ simulator သည် interface procedure လေးခုပါရှိသော Scheme program တစ်ခုဖြစ်သည်။ ပထမတစ်ခုသည် register machine ၏ ဖော်ပြချက်ကို အသုံးပြု၍ machine ၏ model တစ်ခု (စမ်းသပ်မည့် machine ၏ အစိတ်အပိုင်းများနှင့် ကိုက်ညီသော အစိတ်အပိုင်းများပါဝင်သည့် data structure တစ်ခု) ကို တည်ဆောက်သည်။ အခြားသုံးခုမှာ ထို model ကို ကိုင်တွယ်အသုံးပြုခြင်းဖြင့် machine ကို simulate လုပ်ရန် ခွင့်ပြုသည်- 

(make-machine ⟨register-names⟩
              ⟨operations⟩
              ⟨controller⟩)

ပေးထားသော register များ၊ operation များနှင့် controller တို့ပါဝင်သည့် machine ၏ model တစ်ခုကို တည်ဆောက်ပြီး ပြန်ပေးသည်။ 

(set-register-contents! ⟨machine-model⟩ 
                        ⟨register-name⟩ 
                        ⟨value⟩)

ပေးထားသော machine အတွင်းရှိ simulate လုပ်ထားသော register တစ်ခုတွင် တန်ဖိုးတစ်ခုကို သိမ်းဆည်းပေးသည်။ 

(get-register-contents ⟨machine-model⟩
                       ⟨register-name⟩)

ပေးထားသော machine အတွင်းရှိ simulate လုပ်ထားသော register တစ်ခု၏ ပါဝင်သည့်အကြောင်းအရာကို ပြန်ပေးသည်။ 

(start ⟨machine-model⟩)

ပေးထားသော machine ၏ လုပ်ဆောင်ချက်ကို controller sequence ၏ အစမှ စတင်၍ sequence ၏ အဆုံးသို့ ရောက်သောအခါ ရပ်တန့်သည်အထိ simulate လုပ်ပေးသည်။

ဤ procedure များကို မည်သို့အသုံးပြုရသည်ဟူသော ဥပမာအနေဖြင့်၊ 5.1.1 မှ GCD machine ၏ model တစ်ခုအဖြစ် gcd-machine ကို အောက်ပါအတိုင်း သတ်မှတ်နိုင်သည်- 

(define gcd-machine
  (make-machine
   '(a b t)
   (list (list 'rem remainder) (list '= =))
   '(test-b
       (test (op =) (reg b) (const 0))
       (branch (label gcd-done))
       (assign t (op rem) (reg a) (reg b))
       (assign a (reg b))
       (assign b (reg t))
       (goto (label test-b))
     gcd-done)))

make-machine သို့ ပေးရသော ပထမဆုံး argument မှာ register အမည်များ၏ list ဖြစ်သည်။ နောက် argument မှာ operation အမည်တစ်ခုစီကို ထို operation ကို အကောင်အထည်ဖော်သော (ဆိုလိုသည်မှာ တူညီသော input value များပေးလျှင် တူညီသော output value ကို ထုတ်ပေးသော) Scheme procedure နှင့် တွဲပေးထားသည့် table (အစိတ်အပိုင်းနှစ်ခုပါသော list များ၏ list) ဖြစ်သည်။ နောက်ဆုံး argument မှာ 5.1 တွင်ပါရှိသည့်အတိုင်း label များနှင့် machine instruction များ၏ list အဖြစ် controller ကို ဖော်ပြခြင်းဖြစ်သည်။

ဤ machine ဖြင့် GCD များကို တွက်ချက်ရန်၊ ကျွန်ုပ်တို့သည် input register များကို သတ်မှတ်ပေးပြီး၊ machine ကို စတင်ကာ၊ simulation ပြီးဆုံးသောအခါ ရလဒ်ကို စစ်ဆေးရပါမည်- 

(set-register-contents! gcd-machine 'a 206)
done

(set-register-contents! gcd-machine 'b 40)
done

(start gcd-machine)
done

(get-register-contents gcd-machine 'a)
2

ဤတွက်ချက်မှုသည် Scheme ဖြင့် ရေးသားထားသော gcd procedure ထက် ပို၍နှေးကွေးပါလိမ့်မည်။ အကြောင်းမှာ ကျွန်ုပ်တို့သည် assign ကဲ့သို့သော low-level machine instruction များကို ပိုမိုရှုပ်ထွေးသော operation များဖြင့် simulate လုပ်ဆောင်မည်ဖြစ်သောကြောင့်ဖြစ်သည်။

5.2.1စက်ပုံစံငယ်

make-machine မှ ထုတ်လုပ်လိုက်သော စက်ပုံစံငယ် (machine model) ကို Chapter 3 တွင် ဖော်ပြခဲ့သည့် message-passing နည်းပညာများကို အသုံးပြုထားသော local state ရှိသည့် procedure တစ်ခုအဖြစ် ဖော်ပြသည်။ ဤပုံစံငယ်ကို တည်ဆောက်ရန်အတွက် make-machine သည် register machine အားလုံးတွင် တူညီသော စက်ပုံစံငယ်၏ အစိတ်အပိုင်းများကို တည်ဆောက်ရန် make-new-machine procedure ကို စတင်ခေါ်ယူသည်။ make-new-machine မှ တည်ဆောက်လိုက်သော ဤအခြေခံ စက်ပုံစံငယ်သည် အခြေခံအားဖြင့် register အချို့နှင့် stack တစ်ခုအတွက် ထည့်စရာ (container) တစ်ခု ဖြစ်ပြီး၊ ၎င်းနှင့်အတူ controller instruction များကို တစ်ခုပြီးတစ်ခု လုပ်ဆောင်ပေးသော execution mechanism တစ်ခု ပါဝင်သည်။

ထို့နောက် Make-machine သည် ဤအခြေခံပုံစံငယ်ကို (၎င်းထံသို့ message များ ပေးပို့ခြင်းဖြင့်) သတ်မှတ်ထားသော စက်၏ register များ၊ operation များ နှင့် controller တို့ ပါဝင်စေရန် တိုးချဲ့သည်။ ပထမဦးစွာ ၎င်းသည် ပေးထားသော register အမည်တစ်ခုစီအတွက် စက်အသစ်တွင် register တစ်ခု သတ်မှတ်ပေးပြီး (allocate)၊ သတ်မှတ်ထားသော operation များကို စက်ထဲသို့ ထည့်သွင်းပေးသည်။ ထို့နောက် ၎င်းသည် assembler (အောက်ရှိ 5.2.2 တွင် ဖော်ပြထားသည်) ကို အသုံးပြု၍ controller list ကို စက်အသစ်အတွက် instruction များအဖြစ် ပြောင်းလဲပြီး၊ ၎င်းတို့ကို စက်၏ instruction sequence အဖြစ် ထည့်သွင်းပေးသည်။ Make-machine သည် ပြုပြင်မွမ်းမံထားသော စက်ပုံစံငယ်ကို ရလဒ်တန်ဖိုးအဖြစ် ပြန်ပေးသည်။ 

(define (make-machine register-names 
                      ops 
                      controller-text)
  (let ((machine (make-new-machine)))
    (for-each (lambda (register-name)
                ((machine 'allocate-register) 
                 register-name))
              register-names)
    ((machine 'install-operations) ops)
    ((machine 'install-instruction-sequence)
     (assemble controller-text machine))
    machine))
Register များ

ကျွန်ုပ်တို့သည် register တစ်ခုကို Chapter 3 တွင်ကဲ့သို့ပင် local state ရှိသော procedure တစ်ခုအဖြစ် ဖော်ပြပါမည်။ make-register procedure သည် ရယူနိုင်သော သို့မဟုတ် ပြောင်းလဲနိုင်သော တန်ဖိုးတစ်ခုကို ထိန်းသိမ်းထားသည့် register တစ်ခုကို ဖန်တီးပေးသည်- 

(define (make-register name)
  (let ((contents '*unassigned*))
    (define (dispatch message)
      (cond ((eq? message 'get) contents)
            ((eq? message 'set)
             (lambda (value) 
               (set! contents value)))
            (else
             (error "Unknown request: 
                     REGISTER"
                    message))))
    dispatch))

အောက်ပါ procedure များကို register များသို့ ဝင်ရောက်အသုံးပြုရန် အသုံးပြုသည်- 

(define (get-contents register)
  (register 'get))

(define (set-contents! register value)
  ((register 'set) value))
Stack

ကျွန်ုပ်တို့သည် stack တစ်ခုကိုလည်း local state ရှိသော procedure တစ်ခုအဖြစ် ဖော်ပြနိုင်သည်။ make-stack procedure သည် stack ပေါ်ရှိ item များ၏ list တစ်ခုဖြင့် ဖွဲ့စည်းထားသော local state ရှိသည့် stack တစ်ခုကို ဖန်တီးပေးသည်။ Stack တစ်ခုသည် item တစ်ခုကို stack ပေါ်သို့ push လုပ်ရန်၊ stack ထိပ်ဆုံးမှ item ကို pop လုပ်ပြီး ပြန်ပေးရန်နှင့် stack ကို အလွတ်အဖြစ် initialize လုပ်ရန် တောင်းဆိုချက်များကို လက်ခံသည်။ 

(define (make-stack)
  (let ((s '()))
    (define (push x)
      (set! s (cons x s)))
    (define (pop)
      (if (null? s)
          (error "Empty stack: POP")
          (let ((top (car s)))
            (set! s (cdr s))
            top)))
    (define (initialize)
      (set! s '())
      'done)
    (define (dispatch message)
      (cond ((eq? message 'push) push)
            ((eq? message 'pop) (pop))
            ((eq? message 'initialize) 
             (initialize))
            (else 
             (error "Unknown request: STACK"
                    message))))
    dispatch))

အောက်ပါ procedure များကို stack များသို့ ဝင်ရောက်အသုံးပြုရန် အသုံးပြုသည်- 

(define (pop stack) (stack 'pop))
(define (push stack value)
  ((stack 'push) value))
အခြေခံ စက်

ပုံ ၅.၁၃ တွင် ပြထားသော make-new-machine procedure သည် object တစ်ခုကို တည်ဆောက်ပေးပြီး၊ ၎င်း၏ local state တွင် stack တစ်ခု၊ အစပိုင်းတွင် အလွတ်ဖြစ်သော instruction sequence တစ်ခု၊ stack ကို အစပြု (initialize) ပေးသည့် operation တစ်ခု စတင်ပါဝင်သော operation များ၏ list တစ်ခုနှင့် flag နှင့် pc (“program counter” အတွက်) ဟု အမည်ပေးထားသော register နှစ်ခု စတင်ပါဝင်သည့် register table တစ်ခုတို့ ပါဝင်သည်။ အတွင်းပိုင်း procedure ဖြစ်သော allocate-register သည် register table ထဲသို့ entry အသစ်များကို ပေါင်းထည့်ပေးပြီး၊ အတွင်းပိုင်း procedure ဖြစ်သော lookup-register သည် table ထဲတွင် register များကို ရှာဖွေပေးသည်။

ပုံ ၅.၁၃: ↓ အခြေခံ စက်ပုံစံငယ်ကို အကောင်အထည်ဖော်ထားသော make-new-machine procedure။ 

(define (make-new-machine)
  (let ((pc (make-register 'pc))
        (flag (make-register 'flag))
        (stack (make-stack))
        (the-instruction-sequence '()))
    (let ((the-ops
           (list 
            (list 'initialize-stack
                  (lambda () 
                    (stack 'initialize)))))
          (register-table
           (list (list 'pc pc) 
                 (list 'flag flag))))
      (define (allocate-register name)
        (if (assoc name register-table)
            (error 
             "Multiply defined register: " 
             name)
            (set! register-table
                  (cons 
                   (list name 
                         (make-register name))
                   register-table)))
        'register-allocated)
      (define (lookup-register name)
        (let ((val 
               (assoc name register-table)))
          (if val
              (cadr val)
              (error "Unknown register:" 
                     name))))
      (define (execute)
        (let ((insts (get-contents pc)))
          (if (null? insts)
              'done
              (begin
                ((instruction-execution-proc 
                  (car insts)))
                (execute)))))
      (define (dispatch message)
        (cond ((eq? message 'start)
               (set-contents! 
                pc
                the-instruction-sequence)
               (execute))
              ((eq? 
                message 
                'install-instruction-sequence)
               (lambda (seq) 
                 (set! 
                  the-instruction-sequence 
                  seq)))
              ((eq? message 
                    'allocate-register) 
               allocate-register)
              ((eq? message 'get-register) 
               lookup-register)
              ((eq? message 
                    'install-operations)
               (lambda (ops) 
                 (set! the-ops 
                       (append the-ops ops))))
              ((eq? message 'stack) stack)
              ((eq? message 'operations) 
               the-ops)
              (else (error "Unknown request: 
                            MACHINE"
                           message))))
      dispatch)))

flag register ကို simulation ပြုလုပ်ထားသော စက်အတွင်း branching များကို ထိန်းချုပ်ရန် အသုံးပြုသည်။ Test instruction များသည် flag ၏ အကြောင်းအရာကို test ၏ ရလဒ် (မှန် သို့မဟုတ် မှား) အဖြစ် သတ်မှတ်ပေးသည်။ Branch instruction များသည် flag ၏ အကြောင်းအရာကို စစ်ဆေးခြင်းဖြင့် branch လုပ်မလား မလုပ်ဘူးလားဆိုသည်ကို ဆုံးဖြတ်သည်။

pc register သည် စက်လည်ပတ်နေစဉ် instruction များ၏ အစီအစဉ်ကို ဆုံးဖြတ်ပေးသည်။ ဤအစီအစဉ်အတိုင်း လုပ်ဆောင်ခြင်းကို အတွင်းပိုင်း procedure ဖြစ်သော execute ဖြင့် အကောင်အထည်ဖော်ထားသည်။ Simulation model တွင်၊ စက်၏ instruction တစ်ခုစီသည် instruction execution procedure ဟု ခေါ်သော argument မပါသည့် procedure တစ်ခု ပါဝင်သည့် data structure တစ်ခု ဖြစ်ပြီး၊ ထို procedure ကို ခေါ်ယူခြင်းသည် instruction ကို လုပ်ဆောင်ခြင်းကို ပုံတူကူးပေးသည်။ Simulation လည်ပတ်နေစဉ်တွင် pc သည် နောက်ထပ်လုပ်ဆောင်ရမည့် instruction မှ စတင်သော instruction sequence ထဲရှိ နေရာကို ညွှန်ပြနေသည်။ Execute သည် ထို instruction ကို ရယူပြီး၊ instruction execution procedure ကို ခေါ်ယူခြင်းဖြင့် ၎င်းကို လုပ်ဆောင်ကာ၊ လုပ်ဆောင်ရန် instruction မရှိတော့သည်အထိ (ဆိုလိုသည်မှာ pc သည် instruction sequence ၏ အဆုံးကို ညွှန်ပြသည်အထိ) ဤစက်ဝန်းကို ထပ်ခါတလဲလဲ လုပ်ဆောင်သည်။

၎င်း၏ လုပ်ဆောင်ချက် အစိတ်အပိုင်းတစ်ခုအနေဖြင့် instruction execution procedure တစ်ခုစီသည် နောက်ထပ်လုပ်ဆောင်ရမည့် instruction ကို ညွှန်ပြရန် pc ကို ပြုပြင်ပြောင်းလဲသည်။ Branch နှင့် goto instruction များသည် pc ကို ပန်းတိုင်အသစ်သို့ ညွှန်ပြရန် ပြောင်းလဲပေးသည်။ ကျန်ရှိသော instruction အားလုံးသည် sequence ထဲရှိ နောက်ထပ် instruction သို့ ညွှန်ပြစေရန် pc ကို ရှေ့သို့ တိုးပေးရုံသာ လုပ်ဆောင်သည်။ execute ကို ခေါ်ယူတိုင်း execute ကို ထပ်မံခေါ်ယူသည်ကို သတိပြုပါ၊ သို့သော် instruction execution procedure ကို လည်ပတ်ခြင်းက pc ၏ အကြောင်းအရာကို ပြောင်းလဲပေးသောကြောင့် ၎င်းသည် အဆုံးမရှိပတ်လမ်း (infinite loop) ကို မဖြစ်ပေါ်စေပါ။

Make-new-machine သည် အတွင်းပိုင်း state သို့ message-passing ဖြင့် ဝင်ရောက်အသုံးပြုခြင်းကို အကောင်အထည်ဖော်ပေးသည့် dispatch procedure ကို ပြန်ပေးသည်။ စက်ကို စတင်ခြင်းကို pc ကို instruction sequence ၏ အစသို့ သတ်မှတ်ပေးခြင်းနှင့် execute ကို ခေါ်ယူခြင်းတို့ဖြင့် ဆောင်ရွက်ကြောင်း သတိပြုပါ။

5.2.2The Assembler

Assembler သည် စက်တစ်ခုအတွက် controller expression အစီအစဉ်များကို ၎င်းနှင့် သက်ဆိုင်သော machine instruction စာရင်းအဖြစ် ပြောင်းလဲပေးသည်။ instruction တစ်ခုစီတွင် ၎င်း၏ execution procedure ကိုယ်စီ ပါရှိသည်။ ခြုံငုံကြည့်လျှင် assembler သည် Chapter 4 တွင် ကျွန်ုပ်တို့ လေ့လာခဲ့သော evaluator များနှင့် များစွာ ဆင်တူသည်—၎င်းတွင် input language တစ်ခု (ဤနေရာတွင် register-machine language) ရှိပြီး ၎င်း language ရှိ expression အမျိုးအစား တစ်ခုစီအတွက် သင့်လျော်သော လုပ်ဆောင်ချက်ကို ကျွန်ုပ်တို့ လုပ်ဆောင်ရမည် ဖြစ်သည်။

instruction တစ်ခုစီအတွက် execution procedure တစ်ခု ထုတ်လုပ်သည့် နည်းလမ်းသည် 4.1.7 တွင် analysis နှင့် runtime execution ကို သီးခြားစီ ခွဲထုတ်ခြင်းဖြင့် evaluator ၏ အမြန်နှုန်းကို မြှင့်တင်ရန် ကျွန်ုပ်တို့ အသုံးပြုခဲ့သည့် နည်းလမ်းပင် ဖြစ်သည်။ Chapter 4 တွင် တွေ့ခဲ့ရသည့်အတိုင်း Scheme expression များကို ခွဲခြမ်းစိတ်ဖြာရာတွင် variable များ၏ တကယ့် တန်ဖိုးများကို မသိဘဲ များစွာသော အသုံးဝင်သည့် analysis များကို လုပ်ဆောင်နိုင်သည်။ ဤနေရာတွင်လည်း ထို့အတူပင် machine register များ၏ တကယ့် ပါဝင်သည့် အချက်အလက်များကို မသိဘဲ register-machine-language expression များကို အသုံးဝင်သော ခွဲခြမ်းစိတ်ဖြာမှုများ ပြုလုပ်နိုင်သည်။ ဥပမာအားဖြင့် register များကို ရည်ညွှန်းခြင်းများအစား register object များဆီသို့ ညွှန်ပြသော pointer များဖြင့် အစားထိုးနိုင်ပြီး၊ label များကို ရည်ညွှန်းခြင်းများအစား label သတ်မှတ်ထားသော instruction sequence ထဲမှ နေရာသို့ ညွှန်ပြသော pointer များဖြင့် အစားထိုးနိုင်သည်။

instruction execution procedure များကို မထုတ်လုပ်မီ assembler သည် label များအားလုံး ဘာကို ရည်ညွှန်းသည်ကို သိရှိထားရန် လိုအပ်သည်။ ထို့ကြောင့် ၎င်းသည် instruction များမှ label များကို ခွဲထုတ်ရန် controller text ကို scanning ဖတ်ခြင်းဖြင့် စတင်သည်။ text ကို scan ဖတ်နေစဉ်အတွင်း instruction စာရင်း တစ်ခုနှင့် label တစ်ခုစီကို ထိုစာရင်းထဲရှိ နေရာတစ်ခုနှင့် ချိတ်ဆက်ပေးထားသော table တစ်ခုကို တည်ဆောက်သည်။ ထို့နောက် assembler သည် instruction တစ်ခုစီအတွက် execution procedure ကို ထည့်သွင်းခြင်းဖြင့် instruction စာရင်းကို ဖြည့်စွက်ပေးသည်။

assemble procedure သည် assembler ၏ အဓိက entry point ဖြစ်သည်။ ၎င်းသည် controller text နှင့် machine model တို့ကို argument များအဖြစ် လက်ခံပြီး model တွင် သိမ်းဆည်းမည့် instruction sequence ကို ပြန်ပေးသည်။ Assemble သည် ပေးထားသော controller text မှ ကနဦး instruction စာရင်းနှင့် label table ကို တည်ဆောက်ရန် extract-labels ကို ခေါ်ယူသည်။ extract-labels သို့ ပေးရသော ဒုတိယမြောက် argument သည် ရရှိလာသော ရလဒ်များကို လုပ်ဆောင်ရန် ခေါ်ယူရမည့် procedure တစ်ခု ဖြစ်သည်။ ဤ procedure သည် instruction execution procedure များကို ထုတ်လုပ်ရန်နှင့် ၎င်းတို့ကို instruction စာရင်းထဲသို့ ထည့်သွင်းရန် update-insts! ကို အသုံးပြုပြီးနောက် ပြုပြင်ထားသော စာရင်းကို ပြန်ပေးသည်။ 

(define (assemble controller-text machine)
  (extract-labels controller-text
    (lambda (insts labels)
      (update-insts! insts labels machine)
      insts)))

Extract-labels သည် argument များအဖြစ် text စာရင်း (controller instruction expression များ အစီအစဉ်) နှင့် receive procedure တစ်ခုကို လက်ခံသည်။ Receive ကို တန်ဖိုးနှစ်ခုဖြင့် ခေါ်ယူမည် ဖြစ်သည်- (၁) text မှ instruction တစ်ခုစီ ပါဝင်သော instruction data structure များ၏ insts စာရင်း၊ နှင့် (၂) text မှ label တစ်ခုစီကို insts စာရင်းထဲရှိ label ရည်ညွှန်းထားသော နေရာနှင့် ချိတ်ဆက်ပေးသည့် labels ဟု ခေါ်သော table တစ်ခုတို့ ဖြစ်သည်။ 

(define (extract-labels text receive)
  (if (null? text)
      (receive '() '())
      (extract-labels 
       (cdr text)
       (lambda (insts labels)
         (let ((next-inst (car text)))
           (if (symbol? next-inst)
               (receive 
                   insts
                   (cons 
                    (make-label-entry 
                     next-inst
                     insts)
                    labels))
               (receive 
                   (cons (make-instruction 
                          next-inst)
                         insts)
                   labels)))))))

Extract-labels သည် text ၏ element များကို အစဉ်လိုက် scan ဖတ်ပြီး insts နှင့် labels တို့ကို စုဆောင်းခြင်းဖြင့် အလုပ်လုပ်သည်။ အကယ်၍ element တစ်ခုသည် symbol (ထို့ကြောင့် label) ဖြစ်ပါက သင့်လျော်သော entry တစ်ခုကို labels table ထဲသို့ ထည့်သွင်းသည်။ သို့မဟုတ်ပါက ထို element ကို insts စာရင်းထဲသို့ ထည့်သွင်းသည်။289

Update-insts! သည် ကနဦးတွင် instruction များ၏ text သာ ပါဝင်သော instruction စာရင်းကို သက်ဆိုင်ရာ execution procedure များ ပါဝင်စေရန် ပြုပြင်ပြောင်းလဲပေးသည်- 

(define (update-insts! insts labels machine)
  (let ((pc (get-register machine 'pc))
        (flag (get-register machine 'flag))
        (stack (machine 'stack))
        (ops (machine 'operations)))
    (for-each
     (lambda (inst)
       (set-instruction-execution-proc!
        inst
        (make-execution-procedure
         (instruction-text inst) 
         labels
         machine
         pc
         flag
         stack
         ops)))
     insts)))

machine instruction data structure သည် instruction text နှင့် ၎င်းနှင့် သက်ဆိုင်သော execution procedure ကို ရိုးရိုးရှင်းရှင်းပင် တွဲဖက်ပေးထားခြင်း ဖြစ်သည်။ extract-labels က instruction ကို တည်ဆောက်ချိန်တွင် execution procedure ကို မရရှိသေးဘဲ၊ နောက်ပိုင်းတွင်မှ update-insts! က ထည့်သွင်းပေးခြင်း ဖြစ်သည်။ 

(define (make-instruction text)
  (cons text '()))
(define (instruction-text inst) (car inst))
(define (instruction-execution-proc inst)
  (cdr inst))
(define (set-instruction-execution-proc!
         inst
         proc)

5.2.3Instruction များအတွက် လုပ်ဆောင်ချက် လုပ်ငန်းစဉ် (Execution Procedures) များ ထုတ်လုပ်ခြင်း

assembler သည် instruction တစ်ခုအတွက် execution procedure ကို ထုတ်လုပ်ရန် make-execution-procedure ကို ခေါ်ယူပါသည်။ 4.1.7 ရှိ evaluator မှ analyze လုပ်ငန်းစဉ်ကဲ့သို့ပင်၊ ၎င်းသည် သင့်လျော်သော execution procedure ကို ထုတ်လုပ်ရန် instruction အမျိုးအစားအပေါ် မူတည်၍ ခွဲခြားလုပ်ဆောင်ပါသည်။ 

(define (make-execution-procedure 
         inst labels machine pc flag stack ops)
  (cond ((eq? (car inst) 'assign)
         (make-assign 
          inst machine labels ops pc))
        ((eq? (car inst) 'test)
         (make-test 
          inst machine labels ops flag pc))
        ((eq? (car inst) 'branch)
         (make-branch 
          inst machine labels flag pc))
        ((eq? (car inst) 'goto)
         (make-goto inst machine labels pc))
        ((eq? (car inst) 'save)
         (make-save inst machine stack pc))
        ((eq? (car inst) 'restore)
         (make-restore inst machine stack pc))
        ((eq? (car inst) 'perform)
         (make-perform
          inst machine labels ops pc))
        (else (error "Unknown instruction 
                      type: ASSEMBLE"
                     inst))))

register-machine language ရှိ instruction အမျိုးအစားတစ်ခုစီအတွက် သင့်လျော်သော execution procedure ကို တည်ဆောက်ပေးသည့် generator တစ်ခုစီ ရှိပါသည်။ ဤလုပ်ငန်းစဉ်များ၏ အသေးစိတ်အချက်အလက်များသည် register-machine language ရှိ instruction တစ်ခုချင်းစီ၏ syntax နှင့် အဓိပ္ပာယ်ကို သတ်မှတ်ပေးပါသည်။ ကျွန်ုပ်တို့သည် 4.1.2 ရှိ evaluator များတွင် လုပ်ဆောင်ခဲ့သကဲ့သို့ပင်၊ instruction တစ်ခု၏ အစိတ်အပိုင်းများကို ထုတ်ယူရန်နှင့် ခွဲခြားရန် syntax procedures များကို အသုံးပြုခြင်းဖြင့် register-machine expressions များ၏ အသေးစိတ် syntax ကို ယေဘုယျ execution mechanism မှ သီးခြားခွဲထုတ်ရန် data abstraction ကို အသုံးပြုပါသည်။

Assign instruction များ

make-assign လုပ်ငန်းစဉ်သည် assign instruction များကို ကိုင်တွယ်ဖြေရှင်းပေးပါသည်။ 

(define (make-assign 
         inst machine labels operations pc)
  (let ((target 
         (get-register 
          machine 
          (assign-reg-name inst)))
        (value-exp (assign-value-exp inst)))
    (let ((value-proc
           (if (operation-exp? value-exp)
               (make-operation-exp
                value-exp 
                machine
                labels
                operations)
               (make-primitive-exp
                (car value-exp)
                machine
                labels))))
      (lambda ()   ; execution procedure
                   ; for assign
        (set-contents! target (value-proc))
        (advance-pc pc)))))

Make-assign သည် selector များကို အသုံးပြု၍ assign instruction မှ target register အမည် (instruction ၏ ဒုတိယမြောက် အစိတ်အပိုင်း) နှင့် value expression (instruction ကို ဖွဲ့စည်းထားသော စာရင်း၏ ကျန်ရှိသော အစိတ်အပိုင်း) တို့ကို ထုတ်ယူပါသည်။ 

(define (assign-reg-name assign-instruction)
  (cadr assign-instruction))
(define (assign-value-exp assign-instruction)
  (cddr assign-instruction))

register အမည်ကို target register object ထုတ်ပေးရန် get-register ဖြင့် ရှာဖွေပါသည်။ တန်ဖိုးသည် operation တစ်ခု၏ ရလဒ်ဖြစ်ပါက value expression ကို make-operation-exp သို့ ပေးပို့ပြီး၊ မဟုတ်ပါက make-primitive-exp သို့ ပေးပို့ပါသည်။ (အောက်တွင် ဖော်ပြထားသော) ဤလုပ်ငန်းစဉ်များသည် value expression ကို parse လုပ်ပြီး တန်ဖိုးအတွက် execution procedure တစ်ခုကို ထုတ်လုပ်ပေးပါသည်။ ၎င်းသည် value-proc ဟုခေါ်သော argument မပါသည့် လုပ်ငန်းစဉ်တစ်ခုဖြစ်ပြီး၊ simulation ပြုလုပ်နေစဉ်အတွင်း register ထံသို့ သတ်မှတ်ပေးမည့် အမှန်တကယ်တန်ဖိုးကို ထုတ်လုပ်ရန် တွက်ချက်မည်ဖြစ်ပါသည်။ register အမည်ကို ရှာဖွေခြင်းနှင့် value expression ကို parse လုပ်ခြင်းတို့ကို instruction ကို simulation လုပ်တိုင်းတွင် မဟုတ်ဘဲ assembly time တွင် တစ်ကြိမ်သာ လုပ်ဆောင်ခြင်းဖြစ်သည်ကို သတိပြုပါ။ ဤသို့ အလုပ်သက်သာခြင်းသည် ကျွန်ုပ်တို့ execution procedures များကို အသုံးပြုရခြင်း၏ အကြောင်းရင်းဖြစ်ပြီး၊ 4.1.7 ရှိ evaluator တွင် program analysis နှင့် execution ကို သီးခြားခွဲထုတ်ခြင်းဖြင့် ရရှိခဲ့သော အလုပ်သက်သာမှုနှင့် တိုက်ရိုက်သက်ဆိုင်ပါသည်။

make-assign မှ ပြန်ပေးသော ရလဒ်မှာ assign instruction အတွက် execution procedure ဖြစ်ပါသည်။ ဤလုပ်ငန်းစဉ်ကို ခေါ်ယူသောအခါ (machine model ၏ execute လုပ်ငန်းစဉ်မှ ခေါ်ယူခြင်းဖြစ်သည်)၊ ၎င်းသည် target register ၏ အကြောင်းအရာကို value-proc ကို execution လုပ်ခြင်းဖြင့် ရရှိလာသော ရလဒ်သို့ သတ်မှတ်ပေးပါသည်။ ထို့နောက် ၎င်းသည် အောက်ပါလုပ်ငန်းစဉ်ကို လုပ်ဆောင်ခြင်းဖြင့် pc ကို နောက်ထပ် instruction တစ်ခုသို့ တိုးမြှင့်ပေးပါသည်။ 

(define (advance-pc pc)
  (set-contents! pc (cdr (get-contents pc))))

Advance-pc သည် branch နှင့် goto မှလွဲ၍ ကျန်ရှိသော instruction အားလုံးအတွက် ပုံမှန် အဆုံးသတ်ခြင်း ဖြစ်ပါသည်။

Test၊ branch နှင့် goto instruction များ

Make-test သည် test instruction များကို အလားတူပင် ကိုင်တွယ်ပါသည်။ ၎င်းသည် စမ်းသပ်ရမည့် အခြေအနေ (condition) ကို သတ်မှတ်ပေးသည့် expression ကို ထုတ်ယူပြီး ၎င်းအတွက် execution procedure တစ်ခုကို ထုတ်လုပ်ပေးပါသည်။ simulation အချိန်တွင် ထို condition အတွက် လုပ်ငန်းစဉ်ကို ခေါ်ယူပြီး၊ ရလဒ်ကို flag register သို့ သတ်မှတ်ကာ pc ကို တိုးမြှင့်ပေးပါသည်။ 

(define 
  (make-test 
   inst machine labels operations flag pc)
  (let ((condition (test-condition inst)))
    (if (operation-exp? condition)
        (let ((condition-proc
               (make-operation-exp
                condition 
                machine
                labels
                operations)))
          (lambda () 
            (set-contents! 
             flag (condition-proc))
            (advance-pc pc)))

                ASSEMBLE" inst))))

(define (test-condition test-instruction)
  (cdr test-instruction))

branch instruction အတွက် execution procedure သည် flag register ၏ အကြောင်းအရာကို စစ်ဆေးပြီး pc ၏ အကြောင်းအရာကို branch destination သို့ သတ်မှတ်ပေးခြင်း (branch သို့ သွားရန်ဖြစ်ပါက) သို့မဟုတ် pc ကို တိုးမြှင့်ပေးခြင်း (branch သို့ မသွားပါက) ကို လုပ်ဆောင်ပါသည်။ branch instruction တွင် ဖော်ပြထားသော destination သည် label တစ်ခု ဖြစ်ရမည်ဖြစ်ပြီး make-branch လုပ်ငန်းစဉ်က ၎င်းကို အတင်းအကျပ် သတ်မှတ်ထားသည်ကို သတိပြုပါ။ ထို့အပြင် label ကို branch instruction ကို simulation လုပ်တိုင်းတွင် မဟုတ်ဘဲ assembly time တွင် ရှာဖွေထားခြင်းဖြစ်သည်ကိုလည်း သတိပြုပါ။ 

(define 
  (make-branch 
   inst machine labels flag pc)
  (let ((dest (branch-dest inst)))
    (if (label-exp? dest)
        (let ((insts
               (lookup-label 
                labels 
                (label-exp-label dest))))
          (lambda ()
            (if (get-contents flag)
                (set-contents! pc insts)
                (advance-pc pc))))
        (error "Bad BRANCH instruction: 
                ASSEMBLE"
               inst))))

(define (branch-dest branch-instruction)
  (cadr branch-instruction))

goto instruction သည် branch နှင့် ဆင်တူသော်လည်း၊ destination ကို label အဖြစ် သော်လည်းကောင်း၊ register အဖြစ် သော်လည်းကောင်း သတ်မှတ်နိုင်ပြီး စစ်ဆေးရန် condition မရှိခြင်းသာ ကွာခြားပါသည်။ pc ကို အမြဲတမ်း destination အသစ်သို့ သတ်မှတ်ပေးပါသည်။ 

(define (make-goto inst machine labels pc)
  (let ((dest (goto-dest inst)))
    (cond ((label-exp? dest)
           (let ((insts
                  (lookup-label 
                   labels
                   (label-exp-label dest))))
             (lambda () 
               (set-contents! pc insts))))
          ((register-exp? dest)
           (let ((reg
                  (get-register 
                   machine
                   (register-exp-reg dest))))
             (lambda ()
               (set-contents! 
                pc
                (get-contents reg)))))
          (else (error "Bad GOTO instruction: 
                        ASSEMBLE"
                       inst)))))

(define (goto-dest goto-instruction)
  (cadr goto-instruction))

Subexpression များအတွက် Execution procedure များ

reg၊ label သို့မဟုတ် const expression တစ်ခု၏ တန်ဖိုးကို register တစ်ခုသို့ assign လုပ်ရန် (make-assign) သို့မဟုတ် operation တစ်ခုအတွက် input အဖြစ် (အောက်တွင် ဖော်ပြထားသော make-operation-exp) လိုအပ်နိုင်သည်။ အောက်ပါ procedure သည် simulation ပြုလုပ်နေစဉ်အတွင်း ဤ expression များအတွက် တန်ဖိုးများ ထုတ်ပေးရန် execution procedure များကို ထုတ်ပေးသည် -

(define (make-operation-exp
         exp machine labels operations)
  (let ((op (lookup-prim 
             (operation-exp-op exp)
             operations))
        (aprocs
         (map (lambda (e)
                (make-primitive-exp 
                 e machine labels))
              (operation-exp-operands exp))))

                              aprocs)))))

Operation expression များ၏ syntax ကို အောက်ပါအတိုင်း သတ်မှတ်သည် -

Simulation သည် အဆိုပြုထားသော စက်ဒီဇိုင်းတစ်ခု၏ မှန်ကန်မှုကို စစ်ဆေးရန်အတွက်သာမက စက်၏ စွမ်းဆောင်ရည်ကို တိုင်းတာရန်အတွက်လည်း အသုံးဝင်ပါသည်။ ဥပမာအားဖြင့်၊ တွက်ချက်မှုတစ်ခုတွင် အသုံးပြုသည့် stack လုပ်ဆောင်ချက်အရေအတွက်ကို တိုင်းတာသည့် “မီတာ” တစ်ခုကို ကျွန်ုပ်တို့၏ simulation ပရိုဂရမ်တွင် ထည့်သွင်းနိုင်သည်။ ဤသို့လုပ်ဆောင်ရန်အတွက်၊ registers များကို stack ပေါ်တွင် သိမ်းဆည်းသည့် အကြိမ်အရေအတွက်နှင့် stack ၏ အမြင့်ဆုံးရောက်ရှိသွားသော အနက် (maximum depth) တို့ကို မှတ်သားထားနိုင်ရန် ကျွန်ုပ်တို့၏ simulated stack ကို ပြင်ဆင်ပြီး၊ အောက်တွင် ဖော်ပြထားသည့်အတိုင်း စာရင်းအင်းအချက်အလက်များကို ပုံနှိပ်ထုတ်ပေးသည့် message တစ်ခုကို stack ၏ interface တွင် ထည့်သွင်းပါမည်။ ထို့အပြင် make-new-machine ရှိ the-ops ကို အောက်ပါအတိုင်း အစပြု (initialize) ပေးခြင်းဖြင့် stack ၏ စာရင်းအင်းအချက်အလက်များကို ပုံနှိပ်ထုတ်ပေးရန် basic machine model တွင် operation တစ်ခုကို ထည့်သွင်းပါမည်။

ဤသည်မှာ make-stack ၏ ဗားရှင်းအသစ် ဖြစ်သည် - 

(define (make-stack)
  (let ((s '())
        (number-pushes 0)
        (max-depth 0)
        (current-depth 0))
    (define (push x)
      (set! s (cons x s))
      (set! number-pushes (+ 1 number-pushes))
      (set! current-depth (+ 1 current-depth))
      (set! max-depth 
            (max current-depth max-depth)))
    (define (pop)
      (if (null? s)
          (error "Empty stack: POP")
          (let ((top (car s)))
            (set! s (cdr s))
            (set! current-depth
                  (- current-depth 1))
            top)))
    (define (initialize)
      (set! s '())
      (set! number-pushes 0)
      (set! max-depth 0)
      (set! current-depth 0)
      'done)

    (define (print-statistics)
      (newline)
      (display (list 'total-pushes 
                     '= 
                     number-pushes
                     'maximum-depth
                     '=
                     max-depth)))
    (define (dispatch message)
      (cond ((eq? message 'push) push)
            ((eq? message 'pop) (pop))
            ((eq? message 'initialize)
             (initialize))
            ((eq? message 'print-statistics)
             (print-statistics))
            (else
             (error "Unknown request: STACK"
                    message))))
    dispatch))

လေ့ကျင့်ခန်း ၅.၁၅ မှ လေ့ကျင့်ခန်း ၅.၁၉ အထိတွင် register-machine simulator ၌ ထည့်သွင်းနိုင်သည့် အခြားအသုံးဝင်သော စောင့်ကြည့်စစ်ဆေးခြင်းနှင့် အမှားရှာပြင်ခြင်း (monitoring and debugging) လုပ်ဆောင်ချက်များကို ဖော်ပြထားပါသည်။

လေ့ကျင့်ခန်း ၅.၁၄: ပုံ ၅.၁၁ တွင် ပြထားသော factorial စက်ကို အသုံးပြု၍ n ၏ တန်ဖိုးအနည်းငယ် အမျိုးမျိုးအတွက် n ! ကို တွက်ချက်ရန် လိုအပ်သော push အရေအတွက်နှင့် အမြင့်ဆုံး stack depth ကို တိုင်းတာပါ။ သင်ရရှိသော အချက်အလက်များမှတစ်ဆင့် n > 1 ဖြစ်သော မည်သည့် n အတွက်မဆို n ! ကို တွက်ချက်ရာတွင် အသုံးပြုသည့် စုစုပေါင်း push လုပ်ဆောင်ချက်အရေအတွက်နှင့် အမြင့်ဆုံး stack depth တို့အတွက် n ကို အခြေခံသော ဖော်မြူလာများကို ဖော်ထုတ်ပါ။ ၎င်းတို့ တစ်ခုစီသည် n ၏ linear function များ ဖြစ်ကြသဖြင့် ကိန်းသေ (constants) နှစ်ခုဖြင့် သတ်မှတ်နိုင်ကြောင်း သတိပြုပါ။ စာရင်းအင်းအချက်အလက်များကို ပုံနှိပ်ထုတ်ယူနိုင်ရန်အတွက် factorial စက်တွင် stack ကို အစပြုရန်နှင့် စာရင်းအင်းများကို ပုံနှိပ်ရန် ညွှန်ကြားချက်များ (instructions) ကို ထပ်မံဖြည့်စွက်ရပါမည်။ (ပုံ ၅.၄ ရှိ GCD စက်တွင် လုပ်ဆောင်ခဲ့သလိုပင်) n အတွက် တန်ဖိုးကို ထပ်တလဲလဲ ဖတ်ယူခြင်း၊ factorial ကို တွက်ချက်ခြင်းနှင့် ရလဒ်ကို ပုံနှိပ်ခြင်းတို့ ပြုလုပ်နိုင်ရန် စက်ကို ပြင်ဆင်လိုပေမည်။ သို့မှသာ get-register-contents၊ set-register-contents! နှင့် start တို့ကို ထပ်တလဲလဲ ခေါ်ယူရန် မလိုတော့မည် ဖြစ်သည်။

လေ့ကျင့်ခန်း ၅.၁၅: register machine simulation တွင် instruction counting ကို ထည့်သွင်းပါ။ ဆိုလိုသည်မှာ machine model သည် လုပ်ဆောင်ပြီးခဲ့သော ညွှန်ကြားချက် (instructions) အရေအတွက်ကို မှတ်သားထားစေရမည်။ Instruction count တန်ဖိုးကို ပုံနှိပ်ပြီး count ကို သုညသို့ ပြန်လည်သတ်မှတ် (reset) ပေးမည့် message အသစ်တစ်ခုကို လက်ခံနိုင်ရန် machine model ၏ interface ကို တိုးချဲ့ပါ။

လေ့ကျင့်ခန်း ၅.၁၆: instruction tracing ပြုလုပ်နိုင်ရန် simulator ကို အဆင့်မြှင့်တင်ပါ။ ဆိုလိုသည်မှာ ညွှန်ကြားချက်တစ်ခုစီကို မလုပ်ဆောင်မီ simulator သည် ထိုညွှန်ကြားချက်၏ စာသားကို ပုံနှိပ်ဖော်ပြရမည်။ Tracing ကို အဖွင့်အပိတ် ပြုလုပ်နိုင်ရန် trace-on နှင့် trace-off message များကို machine model က လက်ခံနိုင်အောင် ပြုလုပ်ပါ။

လေ့ကျင့်ခန်း ၅.၁၇: လေ့ကျင့်ခန်း ၅.၁၆ ၏ instruction tracing ကို တိုးချဲ့ပါ။ သို့မှသာ ညွှန်ကြားချက်တစ်ခုကို မပုံနှိပ်မီ simulator သည် controller sequence တွင် ထိုညွှန်ကြားချက်၏ အရှေ့တည့်တည့်၌ ရှိနေသော မည်သည့် labels များကိုမဆို ပုံနှိပ်ဖော်ပြမည် ဖြစ်သည်။ ဤသို့လုပ်ဆောင်ရာတွင် instruction counting (လေ့ကျင့်ခန်း ၅.၁၅) ကို မထိခိုက်စေရန် သတိပြုပါ။ လိုအပ်သော label အချက်အလက်များကို သိမ်းဆည်းထားနိုင်ရန် simulator ကို ပြုပြင်ရပါမည်။

လေ့ကျင့်ခန်း ၅.၁၈: Registers များကို trace လုပ်နိုင်ရန် ၅.၂.၁ ရှိ make-register procedure ကို ပြင်ဆင်ပါ။ Registers များသည် tracing အဖွင့်အပိတ်လုပ်ရန် message များကို လက်ခံနိုင်ရမည်။ Register တစ်ခုကို trace လုပ်ထားချိန်တွင် register ထံ တန်ဖိုးတစ်ခု သတ်မှတ်ပေးလိုက်ပါက register ၏ အမည်၊ register ၏ အကြောင်းအရာဟောင်းနှင့် အသစ်သတ်မှတ်လိုက်သော အကြောင်းအရာတို့ကို ပုံနှိပ်ဖော်ပြရမည်။ သတ်မှတ်ထားသော machine registers များအတွက် tracing ကို အဖွင့်အပိတ် လုပ်နိုင်ရန် machine model ၏ interface ကို တိုးချဲ့ပါ။

(set-breakpoint ⟨machine⟩ ⟨label⟩ ⟨n⟩)

(set-breakpoint gcd-machine 'test-b 4)

(proceed-machine ⟨machine⟩)

သူမသည် သတ်မှတ်ထားသော breakpoint တစ်ခုကို အောက်ပါအတိုင်း ပြန်လည်ဖယ်ရှားနိုင်ရမည် - 

(cancel-breakpoint ⟨machine⟩ ⟨label⟩ ⟨n⟩)

သို့မဟုတ် breakpoint အားလုံးကို အောက်ပါအတိုင်း ပြန်လည်ဖယ်ရှားနိုင်ရမည် - 

(cancel-all-breakpoints ⟨machine⟩)

289 ဒီနေရာမှာ receive procedure ကို သုံးတာဟာ extract-labels ကို labels နဲ့ insts ဆိုတဲ့ တန်ဖိုးနှစ်ခုကို သိမ်းဆည်းဖို့ compound data structure တစ်ခု အတိအလင်း မဖန်တီးဘဲ ထိရောက်စွာ ပြန်ပေးနိုင်အောင် လုပ်ဆောင်တဲ့ နည်းလမ်းတစ်ခု ဖြစ်ပါတယ်။ တန်ဖိုးအစုံကို အတိအလင်း ပြန်ပေးတဲ့ အခြား အကောင်အထည်ဖော်မှု တစ်ခုကတော့ 

(define (extract-labels text)
  (if (null? text)
      (cons '() '())
      (let ((result 
             (extract-labels (cdr text))))
        (let ((insts (car result))
              (labels (cdr result)))
          (let ((next-inst (car text)))
            (if (symbol? next-inst)
                (cons 
                 insts
                 (cons 
                  (make-label-entry 
                   next-inst insts) 
                  labels))
                (cons 
                 (cons 
                  (make-instruction next-inst) 
                  insts)
                 labels)))))))

၎င်းကို assemble က အောက်ပါအတိုင်း ခေါ်ယူမှာ ဖြစ်ပါတယ်: 

(define (assemble controller-text machine)
  (let ((result 
         (extract-labels controller-text)))
    (let ((insts (car result))
          (labels (cdr result)))
      (update-insts! insts labels machine)
      insts)))
